# 키퍼의 방 (Keeper's Parlor) — 설치·사용 설명서

**AI 키퍼가 진행하는 1인용 시나리오 RPG 테이블.**
이 문서 하나로 **아무것도 없는 상태 → 플레이**까지, 그리고 모든 기능의 사용법까지 끝낸다.

```
[준비물] → PART A (설치) → PART B (실행·접속) → PART C (설정)
→ PART D (시나리오) → PART E (탐사자) → PART F (세션 플레이) → 업데이트 / 문제 해결
```

---

## 0. 이게 어떻게 도는지 (30초 요약)

```
브라우저/폰 ──▶ http://<서버주소>:4020  (게임 화면)
                    ▼
            keepers-parlor 서버 (Node 프로세스)
                    │ (선택) SillyTavern 폴더를 직접 읽음
                    │        — 커넥션 프로필/API키/캐릭터/페르소나/로어북
                    │ AI 키퍼가 서술 + 서버가 실제 주사위를 굴림
                    ▼
            모든 데이터는 server/data/ 에 저장
```

- **시나리오가 곧 룰이다.** CoC 7판 공포물, 미연시, 육성 시뮬, 추리물 — 시나리오가 정의한 판정·수치·엔딩대로 굴러간다.
- **주사위는 서버가 굴린다.** AI는 판정을 요청만 하고, 진짜 난수 결과를 받아 서술한다.
- **진상은 격리된다.** 시나리오의 스포일러(진상, KP 정보)는 keeper 전용으로 분리되어 플레이어 화면에 절대 나오지 않는다.

---

## 준비물

- **AI API 키** 하나: Gemini 또는 Claude. (SillyTavern을 쓰고 있다면 ST에 넣어둔 키를 그대로 재사용할 수 있어 따로 준비 불필요)
- 돌릴 컴퓨터: 아무 PC(윈도우/맥) 또는 서버(예: 오라클 클라우드 Ubuntu). Node.js 18 이상.
- (선택) SillyTavern — 캐릭터/페르소나 임포트와 API 키 재사용을 쓰려면 서버와 같은 머신에.

# PART A. 설치

## A-1. Node.js (없으면)

Ubuntu:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # v20.x 나오면 OK
```
윈도우/맥: https://nodejs.org 에서 LTS 설치.

## A-2. keepers-parlor 설치

```bash
cd ~
git clone https://github.com/siltax03-crypto/keepers-parlor keepers-parlor    # 또는 압축 받아서 풀기
cd keepers-parlor/server
npm install
```

## A-3. (선택) SillyTavern 연동 — 플러그인·확장 설치

ST와 같은 머신이라면, 파일 몇 개 복사로 ST 화면에서 관리할 수 있게 된다:

```bash
# ST 서버 플러그인 (파일 1개 — 상태/재시작/업데이트 담당)
cp ~/keepers-parlor/st-plugin/index.js ~/SillyTavern/plugins/keepers-parlor.js

# ST 확장 (Extensions 패널의 런처·관리 버튼)
mkdir -p ~/SillyTavern/data/default-user/extensions/keepers-parlor
cp ~/keepers-parlor/st-ext/manifest.json \
   ~/keepers-parlor/st-ext/index.js \
   ~/keepers-parlor/st-ext/style.css \
   ~/SillyTavern/data/default-user/extensions/keepers-parlor/
```

ST `config.yaml`에서 서버 플러그인을 켠다:
```yaml
enableServerPlugins: true
```

폴더 위치가 `~/keepers-parlor`가 아니면 ST 실행 시 환경변수로 알려준다:
```bash
KEEPERS_PARLOR_PATH=/실제/경로/keepers-parlor ./start.sh
```

ST가 `~/SillyTavern`이 아니면 게임 서버 쪽에도 알려준다:
```bash
KEEPERS_PARLOR_ST_PATH=/경로/SillyTavern pm2 start ~/keepers-parlor/server/index.js --name keepers-parlor
```

# PART B. 실행·접속

## B-1. 서버 실행

**PC에서 한 번 써보기:**
```bash
cd keepers-parlor/server
npm start          # → http://localhost:4020
```

**서버에서 상시 운영 (pm2 권장):**
```bash
npm install -g pm2                     # 처음 한 번
pm2 start ~/keepers-parlor/server/index.js --name keepers-parlor
pm2 save
pm2 startup                            # 재부팅 자동 시작 (처음 한 번)
```

## B-2. (외부 접속 시) 포트 열기

폰 등 외부에서 접속하려면 4020 포트를 연다. 오라클 클라우드 기준:

1. 콘솔 → 인스턴스 → VCN → 서브넷 → 보안 목록 → **인그레스 규칙 추가**: 소스 `0.0.0.0/0`, TCP, 포트 `4020`
2. 서버에서:
```bash
sudo iptables -I INPUT -p tcp --dport 4020 -j ACCEPT
sudo netfilter-persistent save
```

> ⚠ 게임 서버에는 로그인이 없다. 주소를 아는 사람은 들어올 수 있으니 함부로 공유하지 말 것.

## B-3. 접속

- 브라우저: `http://<서버IP>:4020` (PC 로컬이면 `http://localhost:4020`)
- ST 연동을 했다면: ST 지팡이(🪄) 메뉴 → **키퍼의 방**, 또는 Extensions 패널의 ⚿ 키퍼의 방 → 🕯 테이블 열기
- 폰: 같은 주소로 접속. **홈 화면에 추가**하면 앱처럼 전체화면으로 열린다.

# PART C. 설정 (⚙ 탭)

| 항목 | 설명 |
|---|---|
| **AI 제공자** | Gemini / Claude — API 키 직접 입력. **ST 프로필** — ST ConnectionManager 프로필 선택 (키는 ST secrets에서 자동으로 읽음. google/vertex/claude/openai/커스텀 프록시 전부 지원) |
| **프리셋 주입** | ST 프로필에 연결된 챗컴플리션 프리셋을 키퍼의 스타일 가이드로 주입. [📋 프리셋 항목 보기/편집]에서 항목별로 여기서만 끄거나(ST 원본 불변), 키퍼 전용 항목을 추가 |
| **나레이션 글꼴 / 글자 크기** | 명조/고딕/테마 기본 + 크기 슬라이더. 이 기기에만 저장, 즉시 적용 |
| **진행 템포** | **대화형**(기본): 한 응답 = 한 장면 박자, 자주 멈춰서 발언권을 줌. **시원시원**: 결정 지점까지 쭉 진행 |
| **서술 문체** | **소설체**(-였다/-했다, 기본) / 경어체(-합니다). 시나리오 원문이 경어체여도 변환해 서술 |
| **🎲 내 판정은 내가 굴린다** | 켜면(기본) 키퍼가 내 탐사자 판정을 요청할 때 멈추고 [판정 굴리기] 버튼이 뜸 |

설정은 서버의 `server/data/settings.json`에 저장된다 (git 밖).

# PART D. 시나리오 (📜 탭)

## 추가하기

- **＋ 시나리오 추가**: 텍스트 붙여넣기, 또는 파일 불러오기 — txt / md / **PDF**(전 페이지 텍스트 추출; 스캔 이미지 PDF는 불가) / **.kp.json**(공유 파일 — 아래 참고)
- 제목을 직접 적으면 분석해도 제목이 안 바뀐다. 비워두면 분석이 자동으로 짓는다.
- 저장 후 **분석하기** — AI가 장면·판정·NPC·엔딩을 구조화하고 진상을 keeper 전용으로 분리한다 (LLM 1회, 30초~수 분). 분석 대기/완료로 목록이 분류된다.

## ✨ AI 작가로 만들기

주제 한 줄(예: "아이돌 데뷔물")만 주면:
1. 작가가 필요한 질문을 한다 (배경·수위·분량·결말 해피/새드 등 — **진상 관련은 절대 묻지 않음**)
2. 갈림길은 실제 주사위로 결정한다 (뭐가 정해졌는지는 비밀)
3. 긴 작품은 플롯을 먼저 설계하고 챕터를 하나씩 집필한다 (📖 n장 완성 표시)
4. 완성본은 **내용을 보여주지 않고** 바로 저장·분석된다 — 자기가 만든 시나리오도 스포 없이 플레이 가능
- 육성/연애물이면 프린세스메이커식 수치 시스템(트래커·마일스톤 게이트·수치 미달 실패 루트)을 설계한다.

## 공유하기

- 시나리오 카드 클릭 → 상세 모달 → **[📤 내보내기]**: 분석본까지 포함된 `.kp.json` 파일 다운로드. ⚠ 파일 안에 진상이 있으니 **플레이할 사람은 열어보지 말 것**.
- 받은 파일은 시나리오 추가 → 파일 선택에서 고르면 끝. 분석본 포함이라 **재분석 없이 바로 플레이 가능**.
- 상세 모달에서 제목 수정, 개요·인원·난이도·분석 보존율(원문 대비 %)도 확인.

# PART E. 탐사자 (🔍 탭)

- **＋ 새 탐사자**: 🎲 자동 생성(직업·특성치·기능 랜덤) 또는 직접 입력. 시트는 언제든 수정 가능 (✏).
- **📷 프사**: 카드의 📷 버튼으로 이미지 업로드 (4MB 이하) — 목록·파티 패널·채팅 말풍선에 표시.
- **🎭 ST에서 가져오기** (ST 연동 시):
  - **페르소나** = 보통 내가 조작할 탐사자 / **캐릭터** = 키퍼가 연기할 동료(KPC)
  - AI가 카드 설명·로어북·CHARM 메모리까지 읽고 **캐릭터성에 맞는 시트**(직업·특성치·기능)를 만든다
  - 여러 개 선택 → 한꺼번에 큐로 임포트 (진행바 표시)
  - 한 카드에 여러 캐릭터가 있으면 각각 만들어져 📁 그룹으로 묶인다 (접기/펼치기)
  - 페르소나에 캐릭터를 연결(커넥션)해뒀으면 목록이 자동으로 필터된다 ([전체 보기] 토글)

# PART F. 세션 플레이 (🕯 탭)

## 세션 만들기

1. **＋ 새 세션** → 시나리오 선택
2. **내 탐사자**(직접 조작) 1명 선택 — 페르소나 출신만 목록에 뜨고, 고르면 동료 목록이 그 페르소나의 커넥션으로 필터됨
3. **동료 탐사자**(키퍼가 연기하는 KPC) 선택 — 안 골라도 됨
4. **이름 확인** — 세션은 한글 이름으로 진행. 영문 이름은 AI가 한글 표기를 제안(Hunter→헌터)하고, 발음이 다르면 고치면 시트 이름도 바뀜
5. 키퍼가 시나리오를 이 파티에 맞게 조정(관계 재설정 — 초면 전제면 카드 관계 무시, 개인 훅 부여)하고 세션이 열린다

## 플레이 화면

| 기능 | 설명 |
|---|---|
| **입력창** | 행동/대사 입력. `이렇게 했다. "대사"` 처럼 쓰면 따옴표 안만 말풍선, 나머지는 나레이션으로 표시. 전송은 ➤ 버튼(또는 Ctrl+Enter), 엔터는 줄바꿈 |
| **🎲 판정 굴리기** | 키퍼가 내 판정을 요청하면 멈추고 버튼이 뜸 — 내가 굴린다 |
| **🔁 밀어붙이기 / 🍀 행운 소모** | 내 판정이 실패하면 선택지가 뜸. 밀어붙이기 = 대가를 걸고 재굴림(실패 시 혹독한 결과), 행운 = 점수를 소모해 실패→성공 전환. [⏭ 그냥 진행]으로 받아들일 수도 |
| **스탯 분배 패널** | 시나리오가 포인트 분배를 요구하면 입력칸 패널이 뜸 (남은 포인트 실시간 표시, 합계 맞아야 확인 가능) |
| **↻ 다시** | 마지막 키퍼 응답을 물리고 재생성. 지시를 적으면("더 짧게", "얘 말투 이상해") 그대로 반영. 그 턴의 상태 변화도 함께 되돌아감 |
| **↩ 되돌리기** | 마지막 턴(내 입력 포함)을 없던 일로 |
| **⏭ 진행 맡기기** | 발언권을 키퍼에게 넘김 |
| **🎲 주사위** | 판정과 무관한 수동 굴림 (1d100, 2d6, 커스텀 식) |
| **파티 패널** | HP/SAN/MP/행운 바, 커스텀 수치 칩(**클릭하면 직접 수정**), 상태이상, 인벤토리, 시트 보기 |
| **극중 오브젝트 패널** | 쪽지·신문·간판·폰 화면 등을 키퍼가 HTML로 직접 만들어 띄움 (애니메이션·인터랙션 포함) |

- **세션 기억**: 긴 세션에서도 오래된 기록을 키퍼가 요약 메모로 압축해 계속 기억한다.
- **광기**: SAN을 한 번에 5+ 잃으면 서버가 룰대로 광기 판정과 발작 테이블을 굴린다.
- 키퍼가 생각하는 중에 창을 닫아도 진행은 서버에서 계속되고 저장된다. 완료/엔딩 본 세션은 목록에서 🏁 그룹으로 접힌다.

# 업데이트할 때

**ST 연동 시 — ST 화면에서:** Extensions → ⚿ 키퍼의 방 → **⬇ 업데이트** (git 최신 받기 + npm install + 플러그인/확장 재복사 + 서버 재시작까지 자동)

**터미널에서:**
```bash
cd ~/keepers-parlor && git pull && npm --prefix server install
cp st-plugin/index.js ~/SillyTavern/plugins/keepers-parlor.js          # ST 연동 시
cp st-ext/* ~/SillyTavern/data/default-user/extensions/keepers-parlor/  # ST 연동 시
pm2 restart keepers-parlor
```

플레이 데이터(`server/data/`)는 git 밖이라 업데이트해도 그대로 남는다.

# 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 접속이 안 됨 | 서버 안 켜짐(`pm2 status`) / 포트 안 열림(B-2) / https로 접속함 → **http://** 로 |
| API 오류 (401/400) | 키 오타·만료 → ⚙ 설정 확인. ST 프로필이면 ST에서 그 프로필로 대화가 되는지 먼저 확인 |
| 분석이 토큰 한도에서 잘림 | 매우 긴 시나리오 — 나눠서 넣거나 모델의 출력 한도가 큰 모델 사용 |
| ST 프로필 드롭다운이 비었음 | ST ConnectionManager에 프로필이 없음 / ST 경로 자동 감지 실패 → `KEEPERS_PARLOR_ST_PATH` 지정 |
| 캐릭터 임포트가 "랜덤 굴림 폴백" | 토스트에 원인 표시됨. 카드에서 텍스트를 못 읽은 경우 `http://서버:4020/api/st/char-debug/카드파일명.png` 로 진단 |
| Extensions에 ⚿ 없음 | 확장 폴더 위치 확인(A-3) / ST 재시작 안 함 |
| 상태 🔴 (서버 꺼짐) | 확장의 🔄 서버 재시작 버튼 또는 `pm2 restart keepers-parlor` |
| 키퍼가 폭주(혼자 쭉 진행) | ⚙ 진행 템포 = 대화형인지 확인, ↻ 다시 + 피드백으로 교정 |
| 모바일에서 화면 확대됨 | 구버전 증상 — 업데이트 후 홈 화면 아이콘 삭제하고 다시 추가 |

- 백업 = 이 폴더 복사. ⚠ `settings.json`에 API 키가 있으니 폴더를 통째로 남에게 주지 말 것.
- 시나리오 공유는 폴더 복사 대신 **내보내기(.kp.json)** 를 쓰면 안전하다.
